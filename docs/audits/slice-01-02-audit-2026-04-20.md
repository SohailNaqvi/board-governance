# ASRB Slice 1 and Slice 2 Audit Report

**Audited by:** Cowork
**Date:** 2026-04-20
**Repository commit:** `32b1489cbcfb0136a37e336014d6a4603a8c8413` (main)

> **Note:** PR #2 (`fix/asrb-slice-01-completion`) adds ComplianceRule/ComplianceEvaluation/RuleEvaluation entities, four domain enums (RuleSource, RuleSeverity, RuleOutcome, ComplianceStatus), a permissions catalog, seed ASRBCase records, and 12 tests. That PR has not been merged at audit time. This report audits what is on `main`.

## Executive Summary

**92 checklist items audited** across Slice 1 (53 items), Slice 2 (33 items), and cross-cutting checks (6 items).

| Status | Count |
|--------|-------|
| PRESENT | 15 |
| PARTIAL | 11 |
| MISSING | 62 |
| INCORRECT | 4 |

**BLOCKS_SLICE_3 findings: 19.** The majority are Slice 1 entity and enum gaps — entire models and state machines that the specification requires but were never created. Several Slice 2 packages and documentation deliverables are also absent. Until the BLOCKS_SLICE_3 items are resolved, Slice 3 cannot proceed.

## Method

Each checklist item from the Slice 1 and Slice 2 briefs was inspected by reading the relevant file in the repository, searching for expected identifiers, and verifying structural properties (model attributes, enum values, function signatures, file existence). Test execution was attempted but blocked by a native-module compatibility issue in the sandbox; test file contents were inspected directly. All findings are based on file inspection against the briefs and ASRB spec v1.0. Prior PR descriptions and commit messages were not treated as evidence.

## Detailed Findings

### DF-1: 12 of 18 Slice 1 Prisma entities are MISSING

**Expected (Slice 1 brief, "Scope of This Slice"):**
> "Extend the Prisma schema with ASRB-specific entities from Section 5 of the ASRB specification: ASRBMeeting, ASRBCase, CaseAttachment, ComplianceRule, ComplianceEvaluation, RuleEvaluation, ASRBAgenda, AgendaBatch, AgendaException, ASRBWorkingPaper, ASRBMember, MemberQuery, CaseDecision, CirculationDecision, RatificationEntry, ExaminerAppointmentStub, VivaSchedulingStub, ResultProcessingStub, CaseAuditEvent."

**Actual:** Only 3 of 18 entities exist: `ASRBCase`, `CaseAttachment`, `CaseAuditEvent`. Additionally, `FeederClient` was added by Slice 2 (not a Slice 1 entity). The remaining 15 entities were never created: ASRBMeeting, ComplianceRule, ComplianceEvaluation, RuleEvaluation, ASRBAgenda, AgendaBatch, AgendaException, ASRBWorkingPaper, ASRBMember, MemberQuery (ASRB), CaseDecision, CirculationDecision, RatificationEntry, ExaminerAppointmentStub, VivaSchedulingStub, ResultProcessingStub.

PR #2 (`fix/asrb-slice-01-completion`) adds 3 of these (ComplianceRule, ComplianceEvaluation, RuleEvaluation), leaving 12 still missing even after that PR merges.

**Severity:** BLOCKS_SLICE_3 (ComplianceRule, ComplianceEvaluation, RuleEvaluation are direct prerequisites). The remaining 9 are MAJOR — required by later slices.

**Recommended remediation:** Create all missing entities in a dedicated remediation PR before Slice 3. Prioritize ComplianceRule, ComplianceEvaluation, RuleEvaluation, and ASRBMeeting first.

### DF-2: 6 of 10 domain enums are MISSING; 4 of 5 state machines are MISSING

**Expected (Slice 1 brief, "Domain Package Additions"):**
> "Enums: ASRBCaseStatus, ComplianceEvaluationStatus, ASRBAgendaStatus, StubStatus, CirculationDecisionStatus, CaseType, CaseOutcome, RuleSource, RuleSeverity, RuleOutcome."
> "Transition tables and canTransition functions for each lifecycle from Section 5.1."

**Actual:** Only ASRBCaseStatus, CaseType, CaseUrgency, and FeederBodyType are present. Missing: ComplianceEvaluationStatus, ASRBAgendaStatus, StubStatus, CirculationDecisionStatus, CaseOutcome, RuleSource, RuleSeverity, RuleOutcome. Only `canTransitionASRBCase` exists; `canTransitionComplianceEvaluation`, `canTransitionASRBAgenda`, `canTransitionStub`, `canTransitionCirculationDecision` are all absent.

PR #2 adds RuleSource, RuleSeverity, RuleOutcome, ComplianceStatus — 4 of the 6 missing enums — but not the state machines.

**Severity:** BLOCKS_SLICE_3 for compliance-related enums and state machine. MAJOR for the rest.

**Recommended remediation:** Add all missing enums and state machines. ComplianceEvaluationStatus and its canTransition are hard prerequisites for Slice 3.

### DF-3: Source-data reader interfaces do not match specification

**Expected (Slice 1 brief, "Source-Data Package Additions"):**
> "Interface IStudentAcademicReader with methods: getStudent(ref), getEnrolmentHistory(ref), getCourseworkStatus(ref), getComprehensiveExamStatus(ref)."
> "Interface ISupervisorProfileReader with methods: getSupervisor(ref), getActiveSupervisionLoad(ref), getPublicationProfile(ref), getHighestQualification(ref)."

**Actual:** `IStudentReader` has `getByRegistrationNumber` only. `ISupervisorReader` has `getByEmployeeId` only. The spec-required multi-method interfaces were never implemented. Mock data has 3 students (spec requires 10) and 3 supervisors (spec requires 5). The `createReader` factory does not include student or supervisor readers.

**Severity:** BLOCKS_SLICE_3. The compliance engine needs detailed student and supervisor data to evaluate rules.

**Recommended remediation:** Replace `IStudentReader`/`ISupervisorReader` with the spec-required interfaces, implement MockReader with 10 students and 5 supervisors, add warehouse stubs, extend factory, add contract tests.

### DF-4: No ASRB-specific roles, no permissions catalog

**Expected (Slice 1 brief, "Role Middleware Extensions"):**
> "Add new permission entries to the centralised permission catalog for ASRB-specific actions (ASRB_VET_CASE, ASRB_APPROVE_AGENDA, ASRB_RECORD_DECISION, ASRB_MANAGE_RULES, ASRB_VIEW_MEMBER_PORTAL, ASRB_ACK_NOTIFICATION)."

**Actual:** The `UserRole` enum contains only Board Governance roles. No ASRB-specific roles exist (Graduate Affairs Officer, Dean Graduate Studies, ASRB Chair, ASRB Member, Downstream Office). No centralized permission catalog. PR #2 adds a `permissions.ts` with `ASRB_MANAGE_RULES` only, but does not add the other 5 permission codes or the ASRB roles.

**Severity:** BLOCKS_SLICE_3. Rule management requires `ASRB_MANAGE_RULES` to be defined and role-gated.

**Recommended remediation:** Extend `UserRole` enum, create all ASRB roles, complete the permissions catalog with all 6 permission codes, and seed users for each role.

### DF-5: @ums/asrb-contracts package does not exist

**Expected (Slice 2 brief, "New Package: @ums/asrb-contracts"):**
> "Location: /packages/asrb-contracts. Contents: zod schemas for envelope, each case_type's case_payload, attachment metadata, response shapes, error shapes. No runtime dependencies on Next.js or Prisma."

**Actual:** Zod schemas are inline in `apps/web/src/lib/asrb/schemas.ts`, tightly coupled to the web app. No standalone package exists. Feeder clients cannot consume the types without depending on the entire web app.

**Severity:** MAJOR. Does not directly block Slice 3 but violates the architectural boundary specified in the brief.

**Recommended remediation:** Extract schemas to `/packages/asrb-contracts` as a workspace package.

### DF-6: @ums/storage package does not exist

**Expected (Slice 2 brief, "New Package: @ums/storage"):**
> "Interface: IAttachmentStorage with methods createPresignedUpload(meta), confirmUpload(id), getDownloadUrl(id, expiresInSeconds), delete(id). Implementation: FilesystemStorage. S3Storage stub."

**Actual:** Upload logic is directly in the route handler (`attachments/[id]/upload/route.ts`) using `writeFileSync`. No interface, no factory, no S3 stub.

**Severity:** MAJOR. Swapping to S3 later would require rewriting the route handler.

**Recommended remediation:** Create `/packages/storage` with the interface and implementations.

### DF-7: API key hashing uses SHA-256 instead of argon2id

**Expected (Slice 2 brief, "Authentication"):**
> "API keys are hashed with argon2id at rest."

**Actual:** `api-key-auth.ts` line 30: `createHash("sha256").update(token).digest("hex")`. SHA-256 is a fast hash, not suitable for secret storage. argon2id is the specified algorithm.

**Severity:** MAJOR for production security; does not block Slice 3.

**Recommended remediation:** Replace SHA-256 with argon2id. Update seed scripts and migration scripts accordingly.

### DF-8: Case-type-specific payload schemas diverge from spec

**Expected (Slice 2 brief, "Case-Type-Specific Payloads"):** Detailed per-type field specifications (e.g., SYNOPSIS_APPROVAL: thesis_title, keywords, similarity_index, defence_date_proposed).

**Actual:** Schemas exist for all 12 types but use different field names and constraints. Key divergences: SYNOPSIS_APPROVAL lacks `similarity_index` (a compliance-critical field); EXAMINER_APPOINTMENT accepts a single examiner instead of an array of 3 with the 2-foreign rule; LEAVE_ABSENCE uses different enum values; COMPREHENSIVE_RESULT uses `performance_level` instead of `overall_outcome`.

**Severity:** MAJOR. Incorrect schemas will cause validation failures when feeders submit spec-compliant payloads.

**Recommended remediation:** Align all 12 schemas to the spec.

### DF-9: No Slice 2 tests exist

**Expected (Slice 2 brief, "Tests"):**
> "Unit tests for: each zod schema, receipt generator, API key verifier, rate limiter, idempotency logic. Integration tests: happy path for each case_type (12 tests); at least 8 distinct rejection paths."

**Actual:** Zero Slice 2 tests. Only Board Governance state-machine tests (17), audit tests (10), and mock-reader contract tests (17) exist.

**Severity:** MAJOR. No automated verification of intake API behavior.

**Recommended remediation:** Write unit tests for each component and integration tests per the brief.

### DF-10: Observability infrastructure is absent

**Expected (Slice 2 brief, "Observability"):**
> "Metrics: asrb_intake_total, asrb_intake_latency_ms, asrb_intake_rejection_total. Structured logs with correlation_id, feeder_client_id, case_type, outcome, latency_ms."

**Actual:** Basic pino logging exists but no correlation IDs, no latency measurement, no metrics counters.

**Severity:** MAJOR for production readiness; does not block Slice 3.

**Recommended remediation:** Add correlation ID middleware, latency timing, and metrics counters.

### DF-11: Reader isolation violated — direct mock instantiation

**Expected (audit brief, "Reader Isolation"):**
> "All source-data access goes through @ums/source-data readers."

**Actual:** `intake-service.ts` lines 127-129 directly instantiate `new MockStudentReader()`, `new MockSupervisorReader()`, `new MockASRBResolutionReader()` instead of using `createReader()`. This hardcodes the mock implementation and bypasses the provider abstraction.

**Severity:** MAJOR. Would fail silently when switching to warehouse mode.

**Recommended remediation:** Use `createReader(process.env.DATA_PROVIDER || 'mock')` and extend factory to include student and supervisor readers.
